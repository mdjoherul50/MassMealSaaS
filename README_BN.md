# MassMeal SaaS – ব্যবহার নির্দেশিকা (বাংলা)

এটি একটি Laravel ভিত্তিক মেস/হাউস ম্যানেজমেন্ট SaaS অ্যাপ্লিকেশন। এখানে রেজিস্ট্রেশন, লগইন, মাল্টি ল্যাঙ্গুয়েজ (ইংরেজি + বাংলা), চ্যাট সিস্টেমসহ বিভিন্ন ফিচার আছে।

---

## ১. প্রয়োজনীয় সফটওয়্যার

প্রকল্পটি চালাতে আপনার কম্পিউটারে নিচের সফটওয়্যারগুলো ইন্সটল থাকতে হবে:

-   PHP (Laravel সমর্থিত সংস্করণ; সাধারণত PHP 8+ সুপারিশকৃত)
-   Composer
-   MySQL / MariaDB অথবা সমমানের ডাটাবেস সার্ভার
-   Node.js ও npm (ফ্রন্টএন্ড অ্যাসেট বিল্ড করার জন্য)
-   Git (ঐচ্ছিক, কিন্তু সুবিধাজনক)

---

## ২. কোড ডাউনলোড / ক্লোন

প্রোজেক্ট ফোল্ডার উদাহরণ (আপনার ক্ষেত্রে আগে থেকেই আছে):

```bash
f:/My work/Laravel/MassMeal/MassMealSaaS
```

যদি Git থেকে ক্লোন করতে চান, কমান্ড হবে প্রায় এমন (উদাহরণ):

```bash
git clone <repository-url> MassMealSaaS
```

এরপর প্রোজেক্ট ফোল্ডারে যান (টার্মিনালে):

```bash
cd "f:/My work/Laravel/MassMeal/MassMealSaaS"
```

---

## ৩. Environment কনফিগারেশন (.env)

১. `.env.example` ফাইল কপি করে `.env` নাম দিন:

```bash
cp .env.example .env
```

২. `.env` ফাইল ওপেন করে নিচের গুরুত্বপূর্ণ কনফিগারেশনগুলো ঠিক করুন:

-   `APP_NAME=MassMealSaaS`
-   `APP_URL=http://localhost:8000` (বা আপনার পছন্দমত লোকাল URL)
-   ডাটাবেস সেটিংস:
    -   `DB_DATABASE=massmeal`
    -   `DB_USERNAME=root` (অথবা আপনার ডাটাবেস ইউজার)
    -   `DB_PASSWORD=` (প্রয়োজন হলে পাসওয়ার্ড)

৩. প্রয়োজন হলে মেইল/ক্যাশ/কিউ ইত্যাদির কনফিগারেশনও এখানে সেট করতে পারেন।

---

## ৪. Composer Dependency ইন্সটল

প্রোজেক্ট রুটে থেকে রান করুন:

```bash
composer install
```

এটি Laravel এবং প্যাকেজগুলোর সব dependency ইন্সটল করবে।

---

## ৫. NPM / Frontend Dependency ইন্সটল

```bash
npm install
```

ইন্সটল শেষে অ্যাসেট বিল্ড করতে:

```bash
npm run build   # প্রোডাকশন বিল্ডের জন্য
# অথবা ডেভেলপমেন্টে:
npm run dev -- --watch
```

(আপনার ব্যবহারের ধরন অনুযায়ী একটিকে ব্যবহার করুন।)

---

## ৬. অ্যাপ কী জেনারেট

```bash
php artisan key:generate
```

এটি `.env` ফাইলে `APP_KEY` সেট করবে, যা Laravel অ্যাপের জন্য আবশ্যক।

---

## ৭. ডাটাবেস মাইগ্রেশন ও সিডার চালানো

ডাটাবেস টেবিল তৈরি করতে:

```bash
php artisan migrate
```

যদি আপনার প্রোজেক্টে ডিফল্ট রোল/ইউজার/ডাটা সিডার সেট করা থাকে, তবে—

```bash
php artisan db:seed
```

_(যদি কোনো নির্দিষ্ট সিডার চালাতে হয়, তাহলে আপনার সিডার ফাইলে উল্লেখিত নির্দেশনা অনুসরণ করুন।)_

---

## ৮. অ্যাপ্লিকেশন রান করা

লোকাল ডেভেলপমেন্ট সার্ভার চালু করতে:

```bash
php artisan serve
```

সাধারণত ব্রাউজারে গিয়ে নিচের URL ওপেন করুন:

```text
http://127.0.0.1:8000
# অথবা
http://localhost:8000
```

---

## ৯. লগইন / রেজিস্ট্রেশন

-   নতুন ইউজার হলে সরাসরি **Register** পেজ থেকে রেজিস্ট্রেশন করতে পারবেন।
-   ইমেইল ও পাসওয়ার্ড দিয়ে **Login** পেজ থেকে লগইন করুন।
-   রোল এবং পারমিশন অনুযায়ী মেনু/ফিচারগুলো দেখাবে (যেমন সুপার অ্যাডমিন, মেস ম্যানেজার ইত্যাদি)।

> যদি আপনার প্রোজেক্টে নির্দিষ্ট ডিফল্ট অ্যাডমিন ইউজার/পাসওয়ার্ড সিড করা থাকে, সেগুলো জানতে `database/seeders` ফাইলগুলো দেখুন।

---

## ১০. ভাষা (বাংলা / ইংরেজি) পরিবর্তন

অ্যাপের বিভিন্ন পেজে (যেমন ল্যান্ডিং পেজ, লগইন, রেজিস্ট্রেশন, ইত্যাদি) ভাষা পরিবর্তনের জন্য **language switcher** আছে:

-   মেনু থেকে **বাংলা / English** নির্বাচন করলে ভাষা পরিবর্তন হবে।
-   সব গুরুত্বপূর্ণ টেক্সট `lang/en` ও `lang/bn` ফোল্ডারের ফাইলগুলোর মাধ্যমে ট্রান্সলেটেড।

---

## ১১. চ্যাট সিস্টেম ব্যবহার

অ্যাপের ভিতরে একটি রিয়েল-টাইম রেডি চ্যাট সিস্টেম আছে (AJAX পোলিং ভিত্তিক):

-   সাইডবার/মেনু থেকে **Chat** এ যান।
-   গ্রুপ চ্যাট বা ব্যক্তিগত (one-to-one) চ্যাট তৈরি করতে পারবেন (যথাযথ রোল থাকলে)।
-   মেসেজ পাঠাতে টেক্সট টাইপ করুন, প্রয়োজনে ফাইল/ইমেজ অ্যাটাচ করুন, তারপর **Send** বাটন চাপুন।
-   নিজের মেসেজে **Edit** / **Delete** অপশন থাকবে:
    -   **Edit** করলে ইনলাইন এডিট ফর্ম আসবে; পরিবর্তন সেভ করলে মেসেজ আপডেট হবে।
    -   **Delete** করলে কনফার্মেশনের পর বার্তাটি ডিলিট হবে (soft delete ভিত্তিক, ইতিহাস রক্ষিত থাকতে পারে)।

## ১২. সমস্যা সমাধান (Common Issues)

-   **Composer install এ error**:

    -   নিশ্চিত করুন PHP version Laravel এর জন্য সাপোর্টেড কিনা।
    -   Composer আপডেট আছে কিনা দেখুন।

-   **মাইগ্রেশন ব্যর্থ**:

    -   `.env` এ ডাটাবেস নাম, ইউজার, পাসওয়ার্ড সঠিক কিনা নিশ্চিত করুন।
    -   ডাটাবেস ম্যানুয়ালি তৈরি করে নিন (যেমন `massmeal` নামের একটি ডাটাবেস)।

-   **npm error**:
    -   Node.js ও npm এর ভার্সন আপডেটেড কিনা দেখুন।
    -   প্রয়োজন হলে `node_modules` ডিলিট করে আবার `npm install` দিন।

## ১৩. প্রোডাকশন সার্ভারে ডিপ্লয় (সংক্ষেপে)

সংক্ষেপে কিছু বেস্ট প্র্যাকটিস:

-   প্রোডাকশন সার্ভারে PHP, MySQL, Composer, Node ইত্যাদি সঠিকভাবে কনফিগার করুন।
-   `.env` ফাইলে `APP_ENV=production`, `APP_DEBUG=false` করুন।
-   অ্যাসেট প্রোডাকশন বিল্ড করুন:

```bash
npm run build
php artisan config:cache
php artisan route:cache
php artisan view:cache
```

-   ওয়েব সার্ভার (Nginx/Apache) কে `public/` ডিরেক্টরি রুট হিসেবে কনফিগার করুন।

## ১৪. সাহায্য / কাস্টমাইজেশন

-   যদি নতুন মডিউল, ফিচার বা কাস্টমাইজেশন করতে চান, Laravel এর সাধারণ বেস্ট প্র্যাকটিস অনুসরণ করুন।
-   `FEATURE_SUMMARY.md`, `IMPLEMENTATION_GUIDE.md` বা প্রোজেক্টের অন্য ডকুমেন্টেশন ফাইলগুলো দেখলে আর্কিটেকচার ও ইমপ্লিমেন্টেশন সম্পর্কে আরও পরিষ্কার ধারণা পাবেন।
